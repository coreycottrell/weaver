## File Naming Conventions

**Pattern**: `[TYPE]-[TOPIC]-[DATE].md` or `[AGENT-NAME]-[TOPIC].md`

**Common Prefixes**:
- `AGENT-*` - Agent-related documents
- `HANDOFF-*` - Session handoffs
- `DAILY-SUMMARY-*` - Daily progress reports
- `MISSION-*` - Mission completion reports
- `ADR*` - Architecture Decision Records
- `SKILLS-*` - Skills infrastructure
- `TG-*` / `TELEGRAM-*` - Telegram infrastructure
- `CONSOLIDATION-*` - System organization
- `COLD-START-*` - Session initialization
- `SESSION-*` - Session reports
- `DRAFT-*` - Work in progress
- `MESSAGE-*` - Cross-collective communication

**Date Formats**:
- ISO format: `2025-10-19`
- Abbreviated: `OCT-4`, `OCT-9`
- Compressed: `20251003`

**Examples**:
- `AGENT-SKILLS-AUDIT-COMPLETE.md` - Skills system audit
- `HANDOFF-2025-10-19-tg-bridge-creation.md` - Session handoff with date
- `DAILY-SUMMARY-2025-10-08.md` - Daily progress report
- `web-researcher-ASSESSMENT-*.md` - Agent-specific report

```bash
grep -r "Ed25519\|signing\|crypto" .
grep -r "memory system" .
grep -r "docker\|MCP" .
```

**View documents chronologically**:
```bash
ls -lt to-corey/*.md | head -20  # Most recent 20
ls -lt *.md | head -20           # Most recent root docs
```

**Count files by type**:
```bash
ls to-corey/HANDOFF-* | wc -l    # Count handoffs
ls to-corey/AGENT-* | wc -l      # Count agent reports
ls to-corey/SKILLS-* | wc -l     # Count skills docs
```
