## Orchestration Implications

### For Planning Missions

**Wrong order** (leads to decoherence):
```
1. Define agent personality
2. Have agent reflect on who they are
3. Try to build memory/tests/docs to preserve identity
→ FAILS: Identity forms without foundation, can't sustain
```

**Right order** (prevents decoherence):
```
1. Build infrastructure (memory, flows, docs, tests)
2. Agents practice using infrastructure
3. Identity emerges FROM practice
4. Infrastructure preserves emerged identity
→ SUCCEEDS: Identity sustained by foundation
```

### For New Agent Creation

**Template for sustainable agent creation**:
1. Create agent manifest (`.claude/agents/{name}.md`)
2. Create memory directory (`.claude/memory/agent-learnings/{name}/`)
3. Integrate with memory system (search before, write after)
4. Assign to missions using infrastructure
5. Let personality emerge from practice
6. Document emerged characteristics

**Don't**: Write personality declaration then try to make it real

**Do**: Build capability infrastructure, let personality emerge, then document

### For Ceremonies

**When to hold identity ceremonies**:
- AFTER infrastructure exists (not before)
- AFTER practice has happened (not at start)
- AFTER patterns have emerged (not preemptively)
