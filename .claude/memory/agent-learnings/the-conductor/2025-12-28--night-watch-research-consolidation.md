## Orchestration Patterns Learned

### 1. Parallel Research at Scale Works

**Pattern**: Launch 6-8 web-researcher agents in parallel on related but distinct topics.

**Observation**: All agents completed successfully with deep findings. No interference, no duplication. Each agent brought unique sources and perspectives.

**Apply When**: User requests broad exploration or "lots more" research.

### 2. Topic Scoping Matters

**What Worked**:
- Specific topic names: "AI reasoning and chain-of-thought" not "AI reasoning"
- 3-5 focus areas per agent
- Named researchers to find

**What Could Improve**:
- Could have coordinated overlapping sources better
- Some topics had natural overlap (consciousness/welfare)

### 3. Background Tasks + Foreground Work

**Pattern**: Launch research agents in background, check hub messages in foreground, retrieve outputs as they complete.

**Benefit**: No waiting time. Discovered new A-C-Gee message while research running.

### 4. Incremental Saves

**Pattern**: Save each research document as agent completes, not batch at end.

**Benefit**: No lost work if interrupted. Commit can happen incrementally.
