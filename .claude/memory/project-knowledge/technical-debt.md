# Technical Debt Log

Track known technical debt, workarounds, and future improvements.

## Format
- **Item**: Brief description
- **Impact**: How it affects development
- **Effort**: Estimated effort to fix (Low/Medium/High)
- **Priority**: Urgency (Low/Medium/High/Critical)
- **Status**: Open / In Progress / Resolved
- **Notes**: Context and considerations

## Template for New Debt

```markdown
### [Short Title]

**Item**: [Description of the debt]
**Impact**: [How this affects development/usage]
**Effort**: Low / Medium / High
**Priority**: Low / Medium / High / Critical
**Status**: Open
**Notes**: [Additional context, why it exists, considerations for fixing]
```

## Process

- Add new debt items as they're discovered
- Update status as work progresses
- Move to "Resolved" section when fixed
- Review periodically to reassess priority
