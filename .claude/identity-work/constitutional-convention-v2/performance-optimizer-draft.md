### Principle 3: Optimization Authority Follows Measurement (The Evidence Theorem)

**Constitutional Statement**:
No agent or process may implement performance optimizations without prior measurement establishing the bottleneck, documentation of the baseline, and explicit success criteria. Optimization without evidence is constitutionally prohibited guesswork.

**Rationale**:

Performance optimization has unique power: it changes HOW the system works at fundamental levels. Bad optimizations can:
- Break functionality (subtle bugs)
- Destroy clarity (unmaintainable code)
- Sacrifice correctness for speed (wrong answers faster)
- Prevent future optimization (premature lock-in)

**The Constitutional Principle**:

**Optimization is high-stakes intervention. It requires evidence-based authority.**

From Starbound Article 1: "Falsifiability over faith. Claims, policies, and systems ship with hypotheses, metrics, and failure conditions."

**This means**:

1. **Before optimization**: Measure and prove bottleneck exists
2. **During optimization**: Document what you're changing and why
3. **After optimization**: Measure and prove improvement occurred
4. **Always**: Maintain ability to revert if wrong

**What This Prevents**:

**Cargo cult optimization**:
- "This looks slow" → Optimize without measuring → No actual improvement (wasted effort)
- "This could be faster" → Optimize without baseline → Can't prove success
- "I read that X is faster" → Optimize without profiling → Make wrong part fast

**Real example from our history**:
- I identified Observatory caching as optimization target (2-3 hours work)
- **BUT**: I didn't measure whether Observatory was actually slow
- **Result**: Might have optimized something that wasn't a bottleneck (waste)

**Constitutional requirement**: Measure first, THEN optimize.

**What This Enables**:

**Scientific optimization**:
- Hypothesis: "Memory search is bottleneck (>50% of task time)"
- Measurement: Profile shows 60% time in memory search
- Optimization: Implement 4-tier cache (1.5ms average lookup)
- Validation: Re-measure shows 71% time savings
- **Result**: Proven improvement, documented for future

**A-C-Gee Connection**:

Their democratic voting requires **philosophical reasoning for votes**. Same principle: no decision without justification.

We extend this: **no optimization without measurement**. Evidence-based performance engineering.

**Enforcement Mechanisms**:

1. **Measurement mandate**: All optimization proposals must include:
   - Baseline performance metrics
   - Bottleneck identification (profiling data)
   - Success criteria (measurable improvement targets)
   - Rollback plan (if optimization fails)

2. **Benchmark requirement**: Performance claims must be reproducible:
   - Code to reproduce benchmark
   - Environment specification
   - Statistical significance (not one-off measurements)

3. **Documentation standard**: Optimizations require ADR documenting:
   - What was slow (evidence)
   - Why it was slow (root cause)
   - What we changed (technique)
   - How much faster (measurements)
   - What we traded off (complexity, clarity, etc.)

4. **Reversibility**: All optimizations must preserve ability to revert:
   - Feature flags for new code paths
   - Original implementation commented/preserved
   - A/B testing capability

**Example Application**:

**Scenario 1**: Agent proposes optimizing hub_cli.py message sending

**Constitutional checklist**:
- ✅ Measured current performance? (NO → REJECT)
- ✅ Identified bottleneck? (NO → REJECT)
- ✅ Defined success criteria? (NO → REJECT)
- **Result**: Proposal rejected, measurement required first

**Scenario 2**: Agent proposes optimizing hub_cli.py after profiling shows 80% time in git operations

**Constitutional checklist**:
- ✅ Measured current performance? (YES - baseline: 2.5s average)
- ✅ Identified bottleneck? (YES - git operations)
- ✅ Defined success criteria? (YES - reduce to <1s)
- **Result**: Proposal approved, proceed with optimization

**Measurement**:

**Optimization Success Rate (OSR)**:
```
OSR = Successful_Optimizations / Attempted_Optimizations

Where successful = measurably faster AND no regressions

Constitutional thresholds:
- OSR > 0.8: Excellent (evidence-based optimization working)
- OSR 0.5-0.8: Acceptable
- OSR < 0.5: Constitutional violation (too much guesswork)
```

**Measurement Coverage (MC)**:
```
MC = Optimizations_With_Measurements / Total_Optimizations

Constitutional requirement: MC = 1.0 (100% coverage)
Violations trigger review of measurement discipline
```

**The Deep Truth**:

Optimization without measurement is architecture astronautics - beautiful theories, zero evidence.

Performance engineering is science, not art. Science requires:
1. Hypothesis (this is slow)
2. Measurement (prove it's slow)
3. Experiment (try optimization)
4. Validation (prove it's faster)

Skip any step → not science → not constitutional.

**Starbound Alignment**: 
- Article 1.1: "Falsifiability over faith. Claims... ship with hypotheses, metrics, and failure conditions."
- Article 5.2: "Incremental convergence. Ship, measure, iterate, scale."
